Q: Como posso criar e submeter objectos digitais ao RODA

Tem à sua disposição 3 vias para criar e submeter objectos digitais ao RODA:

  • SIP com criação e submissão online: utilizar as facilidades disponíveis na página Submeter SIP – solução recomendada apenas para uso esporádico (poucos SIP e de pequena dimensão)
  • SIP com criação off-line e submissão online: descarregar a ferramenta RODA-in que auxilia o processo de criação de SIP e permite a sua submissão electrónica – solução recomendada para a generalidade dos casos, a evitar apenas no componente de submissão, quando perante um SIP de dimensão superior a 2GB.
  • SIP com criação e submissão off-line: descarregar a ferramenta RODA-in para auxílio ao processo de criação do SIP, depois gravado em suporte externo (CD-R, DVD) e entregue presencialmente ou por via postal na DGARQ – solução recomendada para SIP com dimensão superior a 2GB.

Q: Que necessito para utilizar a aplicação RODA IN?

É necessário ter a aplicação Java (versão 1.5 ou posterior) instalada, para poder correr a aplicação RODA IN. A aplicação Java pode ser descarregada em http://www.java.com/en/download/manual.jsp

Q: Como utilizo o RODA IN para submeter SIP

  1. Ao seleccionar Actualizar Plano é pedida a validação do login e palavra-Chave.
  2. Seleccionar o nome de um elemento da hierarquia do plano permite visualizar a metainformação descritiva relativa ao referido elemento.
  3. Produzir SIP
    1. Seleccionar Criar
    2. Introduzir metainformação e selecionar entre Documento composto e Documento Simples
    3. (Criar Documento Simples/Remover Documento Simples)
    4. Seleccionar Criar Representação
    5. Seleccionar a classe da representação pretendida:
      1. Texto com Estrutura (PDF, Microsoft Word, Rich Text Format)
      2. Imagens com Estrutura (TIFF, JPEG, GIF, PNG, BMP)
      3. Bases de Dados Relacionais (MySQL, PostgreSQL, Microsoft SQL Server, Microsoft Access, DBML)
      4. Áudio (MP3, WAV, WMA)
      5. Vídeo (MPEG, AVI, WMV)
      6. Desconhecido – tipo genérico. Sem garantia de preservação
  4. O ecrã das Representações permite seleccionar e organizar as representações
    1. Pode criar novas secções
    2. Pode remover secções
    3. Pode adicionar representação
    4. Pode remover representação
    5. Pode mover a representação para cima ou para baixo de outras.
    6. Seleccionar numa representação permite a sua visualização na área de Pré-Visualização
  5. Seleccionar Guardar
  6. Seleccionar Validar para verificar se o pacote é válido
  7. Seleccionar Enviar
  8. Em Carregar Pacotes, pode:
    1. Escolher os pacotes a ingerir
    2. Escolher o modo de envio
      1. Online
      2. Offline, por correio

Q: Como utilizo o WebRODA para submeter SIP

Existem duas opções de submissão de objectos Digitais

  • Solução para Enviar Pacote (recomendada apenas para uso esporádico, i.e., poucos SIP e de pequena dimensão)
    1. Em Ingestão seleccione em Enviar
    2. No painel Enviar Pacote, seleccione essa opção
    3. Clique no botão Browse para seleccionar a representação que deseja adicionar
    4. Seleccione adicionar
    5. Caso queira retirar uma das representações, clique no sinal “-“ que surge à direita do nome da representação
    6. Seleccione Submeter. Quando a Submissão estiver concluída o ecrã vai directamente para o Painel Estado
  • Solução para Criar Pacote
    1. Em Ingestão seleccione em Enviar
    2. No painel Criar Pacote, seleccione essa opção
    3. Preencha o formulário da Metainformação
    4. Seleccione o tipo de representações que pretende adicionar
    5. Clique no botão Browse para seleccionar a representação que deseja adicionar
    6. Seleccione adicionar
    7. Caso queira retirar uma das representações, clique no sinal “-“ que surge à direita do nome da representação
    8. Seleccione a referência onde pretende integrar o SIP, na tabela Escolha o destino.
    9. Seleccione Submeter. Quando a Submissão estiver concluída o ecrã vai directamente para o Painel Estado

Q: Posso submeter qualquer representação para o RODA?

Antes de criar e submeter pacotes de informação ao RODA, tenha presente que, nos termos da política de preservação digital do RODA, é rejeitada a submissão de pacotes que:

  • Não estejam previstos no Plano de aquisição negociado;
  • Se apresentem em formatos não declarados aceitáveis pelo RODA ou não especificamente autorizados no Acordo de aquisição;
  • Tenham, em algum dos seus ficheiros constitutivos, vírus detectado pelo RODA;
  • Se apresentem com falhas ou erros na metainformação associada.

Q: Onde posso verificar o Estado das submissões de SIPs?

O Painel Estado permite verificar o nome, a data, o estado do pacote submetido, o progresso do estado (medido em percentagem) e o nome do produtor.

Q: Como posso visualizar apenas SIPs pertencentes a uma certa categoria?

É possível Filtrar os processos, de forma a visualizar apenas os que se encontram numa destas categorias:

  • Concluídos
  • A processar
  • Aceites
  • Rejeitados
  • Todos
  • Produzidos por um produtor, devendo-se escrever o nome do produtor pretendido.

Q: Como posso obter uma lista de pacotes em fase de submissão?

É possível Exportar a lista actual para PDF ou para CSV (formato de tabela)

Q: Como posso obter informação sobre as diversas fases da ingestão?

O Relatório de Ingestão apresenta o nome do pacote submetido e o estado por que já passou.

Estas fases são:

  1. Pacote recebido - Carregar pacote para servidor
  2. Pacote Descomprimido – Descompactar o pacote
  3. Livre de Vírus – verificação com antivírus
  4. Envelope válido – verificação de coerência (checksums, organização)
  5. Autorizado – Verificação de autorização de ingestão pró parte do produtor
  6. Ingerido – Representações e metainformação integrados no RODA
  7. Concluído – Ingestão concluída.
  8. Aceite – o SIP foi aceite por quem tem permissões para o aceitar, ou
  9. Rejeitado – O SIP foi rejeitado por quem tem permissões para tal. A Razão de tal rejeição surge no elemento Detalhes.

A metainformação sobre cada uma destas fases inclui:

  • Data e Hora: em que ocorreu a fase
  • Tarefa: Indicação da operação
  • Resultado: pode ser Sucesso ou Falha. O último caso o processo de ingestão pára.
  • Detalhes: operações que ocorrem dentro de cada fase

Q: Como posso aceder à representação e a sua metainformação em processo de ingestão

A Visualização permite aceder à Metainformação descritiva e de Preservação, visualizar as representações e as permissões de clientes, após o processamento ter terminado.

No menu de Descrição é possível:

  • Editar a Metainformação
  • Verificar subníveis (caso existam)
  • Mover as representações do pacote para as integrar noutro destino.
  • Descarregar a descrição em EAD no formato XML

No menu Visualização pode-se:

  • Descarregar a representação original
  • Descarregar a representação normalizada
  • Pré-visualizar a representação no visualizador de fotos (quando possível)
  • Pré-visualizar a representação no visualizador de livros (quando possível)

No menu Preservação é possível:

  • Visualizar a calendarização dos eventos de preservação e respectiva metainformação de preservação da representação.
  • Seleccionar a opção de visualização de metainformação de preservação por Dia e por Mês
  • Descarregar a descrição em PREMIS no formato XML

No menu Permissões pode-se:

  • Visualizar utilizadores e grupos de utilizadores
  • Visualizar e editar as permissões de acesso dos utilizadores e grupos de utilizadores relativamente às representações do pacote submetido.
    As permissões possíveis são:
    • Sem Acesso
    • Acesso a metainformação e disseminações
    • Acesso e edição de metainformação
    • Controlo completo
  • Aplicar a permissão escolhida, de forma recursiva, a todos os sub-níveis
  • Adicionar Utilizador
  • Adicionar Grupo de utilizadores
  • Guardar Alterações

Q: Como posso validar a representação e respectiva metainformação

O Botão de Aceitar permite confirmar que tanto a representação, como a metainformação estão correctas e que estão em condições de serem acedidos no catálogo pelos utilizadores com permissão para tal.

Q: Como posso rejeitar a representação e respectiva metainformação

O Botão de rejeitar tem como objectivo a rejeição de um pacote, permitindo:

  • Escrever uma mensagem explicativa das razões pelas quais o SIP foi rejeitado
  • Escolher mensagens padrão
  • Notificar o produtor
  • Rejeitar – confirmar a decisão de rejeição
  • Cancelar – Não rejeitar.

Q: Que formatos são aceites para submissão?

Os formatos aceites integram-se actualmente nas categorias de Texto, Imagem fixa, Audio, Vídeo.

Texto:

  • Portable Document File (.pdf)
  • Microsoft Word.Document (doc, .docx)
  • Rich Text File (.rtf)
  • Open Document Text (.odt)

Imagem:

  • eXtensible Markup Language (.xml)
  • Joint Photographic Experts Group File (.jpg, jpeg)
  • Graphics Interchange Format (.gif)
  • Tagged Image File Format (.tiff ,.tif)
  • Portable Network Graphics (.png)

Aúdio:

  • MPEG-1 Audio Layer I (.mp1)
  • MPEG-1 Audio Layer II (.mp2)
  • MPEG-1 Audio Layer III (.mp3)
  • Waveform audio format (.wav)
  • Audio Interchange File Format (.aif .aiff)
  • Ogg Vorbis audio file format (.ogg)

Vídeo:

  • Moving Picture Experts Group format file (.mpg, .mpeg)
  • Audio Video Interleave (.avi)
  • Windows Media Video (.wmv)
  • QuickTime file format (.mov)

Q: Posso submeter outros formatos que não integrem a lista anterior

A submissão de outros formatos pode ser aceite desde que estejam expressos no Acordo de custódia. No entanto, o RODA poderá não garantir a preservação da acessibilidade dos mesmos.

Q: Quais os sistemas de gestão de bases de dados que actualmente o RODA pode ingerir informação neles constantes

MySQL, PostgreSQL, Microsoft SQL Server, Microsoft Access, e quaisquer outros cuja informação seja passível de ser convertida para o formato DBML

Q: Como posso esclarecer dúvidas sobre o Acordo de custódia, e/ou actualizar o Plano de aquisição

Em caso de dúvida sobre os termos do Acordo de aquisição celebrado e, mais precisamente, sobre o Plano de transferência negociado, utilize os contactos oficiais para solicitar quaisquer esclarecimentos à DGARQ. Utilize a mesma via para manifestar eventuais necessidades de actualizar o Plano de aquisição previamente negociado.

Caso necessite de obter formação específica sobre os procedimentos de ingestão, contacte-nos com o seu pedido de formação.
