Pre-ingest instructions

The pre-ingest process depicts the ability of a Producer to create Submission Information Packages (SIP) containing both data and metadata (in a well-defined structure) in order to submit them to the repository for ingest. The SIPs created are expected to comply to the policies established by (or negotiated with) the repository. The pre-ingest process usually comprises some or all of the following activities:

1. Negotiation

This activity consists of the definition of the terms, pre-conditions and requirements for content, and accompanying information (e.g. metadata, documentation, contracts, etc.), to be sent to the repository by the Producer. Part of this activity consists of the creation of a base classification scheme (or list of base collections) on which the Producer may deposit new informational items.

2. Submission agreement

This activity consists of signing a written agreement between the Producer and the Repository that specifies the type of content and all the legal and technical requirements that both parties are expected to comply.

3. Submission Information Packages (SIP)

This activity consists of the preparation of one or more Submission Information Packages (SIP) according to the technical and non-technical requirements as defined by the Submission Agreement. To facilitate the creation of SIPs, Producers may take advantage of the RODA-in tool. The tool and its documentation are available at http://rodain.roda-community.org

4. Transfer of materials

This activity consists of the electronic transfer of Submission Information Packages (SIP) from the Producer to the Repository. SIPs are temporarily stored on a quarantine area waiting to be processed by the repository. If changes have been made to the classification scheme externally, e.g. by using a tool like RODA-in, the new classification scheme must be uploaded to the repository previous to any ingest activity.

There are several ways Producers may send SIPs to the repository. These include, but are not limited to the following options:

Ingest process

After transfer, SIPs will be selected for ingest by the Archive. The Ingest process provides services and functions to accept Submission Information Packages (SIPs) from Producers and prepare the contents for storage and management within the Archive.
