Auditprotokolle sind spezielle Dateien, die wichtige Ereignisse aufzeichnen, die im Repository auftreten. Zum Beispiel wird jedes Mal ein Eintrag gemacht, wenn sich ein Benutzer anmeldet, wenn ein Download durchgeführt wird oder wenn eine Änderung an einer beschreibenden Metadatendatei gespeichert wird. Wann immer diese Ereignisse eintreten, zeichnet das Repository die notwendigen Informationen im Ereignisprotokoll auf, um eine künftige Prüfung der Systemaktivität zu ermöglichen. Für jedes Ereignis werden folgende Informationen aufgezeichnet: Datum, beteiligte Komponente, Systemmethode oder Zielobjekte, Benutzer, Dauer der Aktion und IP-Adresse des Benutzers. Die Benutzer können Ereignisse nach Typ, Datum und anderen Attributen filtern, indem sie durch Auswahl der im rechten Bereich verfügbaren Optionen.
