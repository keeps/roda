Välkommen till RODA!

Ett e-arkiv baserat på open-source

RODA är ett system för digitalt bevarande, ett e-arkiv, som levererar full funktionalitet i enlighet med OAIS-modellen. RODA hanterar inleverans, vård och tillgängliggörande av digitala handlingar. RODA är baserat på open-source och supporterar förutom Open Archival Information System (OAIS) även Metadata Encoding and Transmission Standard (METS), Encoded Archival Description (EAD), Dublin Core (DC) and PREMIS (Preservation Metadata).

Överensstämmer med öppna standarder

RODA följer den öppna standarden EAD för beskrivande metadata, PREMIS för bevarandemetadata, METS för strukturell metadata och flera andra standarder för teknisk metadata (exempelvis NISO Z39.87 för bilder).

Leverantörsoberoende

RODA är 100% baserat på open-source. Infrastrukturen för att drifta RODA är helt leverantörsoberoende. Det betyder att det går att använda hårdvara och Linux-distributioner enligt egna önskemål.

Skalbart

RODAS serviceorienterade arkitektur gör att systemet är högst skalbart och möjliggör fördelning av processbelastningar mellan flera servrar. Dessutom gör RODAs avancerade indexkomponenter det möjligt för systemets sök- och visningsfunktioner att spridas mellan flera servrar i ett kluster för ännu bättre prestanda.

Inbyggda funktioner för arkivvård

Arkivvårdsjobb och annan administrering i RODA hanteras av jobbexekveringsmodulen. Modulen låter systemet köra arkivvårdsjobb på utvald information. Arkivvårdsjobben inkluderar formatkonvertering, verifiering av checksummor, rapportering (t. ex. skicka e-post kring godkänd/avvisad SIP), viruskontroller med mera.

Autencitet

RODA använder bevarande metadata (PREMIS) för att skapa en förtroendekedja mellan alla formatmigreringar och innehållsverifieringar. Bevarandemetadata tillsammans med upprättandet av förtroende för dess omgivande miljö (ISO 16363) säkerställer tillförlitligheten av tjänsten och äktheten av de bifogade digitala dokumenten.

Hanterar alla typer av format

Det är möjligt att leverera in alla möjliga sorters information till RODA. Arkivvårdsjobben för konvertering och migrering kan hantera textdokument, bildformat med raster, relationsdatabaser, video- och ljudfiler samt funktion för att konvertera filer till arkivbeständiga format. Det finns plug-ins som möjliggör att RODA enkelt kan stödja konvertering till ytterligare format.

Hanterar och kan anpassas till nya tekniker

Plug-in- och jobbexekveringsmodulen möjliggör ett enkelt sätt att lägga till mer funktionalitet till systemet (t.ex. nya bevarande åtgärder, varningar, verktyg etc.). Dessutom gör RODAs serviceinriktade arkitektur att dess komponenter stegvis kan uppdateras, vilket möjliggör en heterogen användning av teknik på alla nivåer: hårdvara, OS och applikationer.

Avancerad behörighetsstyrning

Användare måste autentiseras innan de kommer åt e-arkivets innehåll. Alla användares aktiviteter loggas i systemet för full spårbarhet. Behörighetsstyrningen är granulär och kan styras ner på enskilda arkivobjekt.

Integrationer med tredjepartsprogram

RODA tillgängliggör all sin funktionalitet via väldokumenterade REST-API:er. Lämpliga Java-bibliotek finns tillgängliga på GitHub för alla utvecklare som interagerar med RODA via dess Core APIs. Det finns ett flertal verktyg som skapar, transformerar och leverarar SIP:ar till RODAs inleveransprocess.

Kvalificerad process för inleverans

RODA stödjer både inleverans av nytt digitalt material så väl som tillhörande metadata i flera olika format. För att skapa SIP:ar enligt de format som stöds finns verktyg tillgängliga för användarna. Arbetsflödet för inleverans kan anpassas i användargränssnittet för att anpassas till särskilda policyer och data.

Och mycket mer...

RODA (Repository of Authentic Digital Records) är ett digitalt arkivsystem för långtidsbevarande av information. Systemet implementerar ett arbetsflöde för inleverans som inte bara validerar SIP:ar utan även skannar dess innehåll efter virus, identifierar format och versioner, extraherar teknisk metadata och konverterar filer till mer arkivbeständiga format.

RODA ger även tillgång till digital information i flera former genom exempelvis sök- och navigeringsmöjligheter av tillgänglig metadata såväl som visualisering, nedladdning av informationen i original, bevarandeformat och spridningsderivat (DIP:ar) direkt i webbgränssnittet.

Administrationsgränssnittet tillåter användare med högre behörighet att hantera och adminsitrera arkivbestånd/informationssamlingar samt definiera regler för bevarandeåtgärder och arkivvårdsjobb. All interaktion mellan användare (människa och maskin) och systemet loggas av säkerhets- och ansvarsskäl.

RODA säkerställer att inlevererad data är autentisk genom att skapa PREMIS-metadata för alla åtgärder som utförs av systemet, spara information om datans ursprung i arkivmetadatastandarder som EAD eller DC och säkerställer integritet och tillgänglighet genom att ofta övervaka data samt se till att den inte har manipulerats.
