Välkommen till RODA!

Ett e-arkiv baserat på open-source

RODA är ett system för digitalt bevarande, ett e-arkiv, som levererar full funktionalitet i enlighet med OAIS-modellen. RODA hanterar inleverans, vård och tillgängliggörande av digitala handlingar. RODA är baserat på open-source och supporterar förutom Open Archival Information System (OAIS) även Metadata Encoding and Transmission Standard (METS), Encoded Archival Description (EAD), Dublin Core (DC) and PREMIS (Preservation Metadata).

Överensstämmer med öppna standarder

RODA följer den öppna standarden EAD för beskrivning av metadata, PREMIS för bevarandeinformation, METS för metadatasturktur och flera andra standarder för teknisk metadata (exempelvis NISO Z39.87 för bilder).

Leverantörsoberoende

RODA är 100% baserat på open-source. Infrastrukturen för att drifta RODA är helt leverantörsoberoende. Det betyder att det går att använda hårdvara och Linux-distributioner enligt egna önskemål.

Skalbart

RODAS serviceorienterade arkitektur gör att systemet är högst skalbart och möjliggör fördelning av processbelastningar mellan flera servrar. Vidare gör RODAs avancerade indexkomponenter det möjligt att systemets sök- och visningsfunktioner kan balanseras mellan flera servrar i ett kluster för ännu bättre prestanda.

Inbyggda funktioner för arkivvård

Arkivvårdsjobb och administrering inom RODA hanteras genom en processmodul. Modulen låter arkivvårdsjobb att köras på specifik utvald information. Arkivvårdsjobben inkluderar formatkonvertering, verifiering av checksummor, rapportering (t. ex. skicka e-post kring godkänd/avvisad SIP), viruskontroller med mera.

Autencitet

RODA använder bevarande metadata (PREMIS) för att skapa en förtroende kedja mellan alla formatmigreringar och innehållsverifieringar. Bevarande metadatat, tillsammans med upprättandet av förtroende för dess omgivande miljö (ISO 16363) säkerställer tillförlitligheten av tjänsten och äktheten av de bifogade digitala dokumenten.

Support för flera format

Det är möjligt att leverera in flera olika typer av information i RODA. Processen för inleverans kan hantera textdokument, bildformat med raster, relationsdatabaser, videofiler och ljudfiler samt funktion för att konvertera dem till arkivbeständiga format. Det finns plug-ins som möjliggör att RODA enkelt kan stödja konvertering till ytterligare format.

Hanterar och anpassas till nya tekniker

Plug-in- och jobbexekveringsmodulen möjliggör ett enkelt sätt att lägga till mer funktionalitet till systemet (t.ex. nya bevarande åtgärder, varningar, verktyg etc.). Dessutom gör RODAs serviceinriktade arkitektur att dess komponenter stegvis kan uppdateras, vilket möjliggör en heterogen användning av teknik på alla nivåer: hårdvara, OS och applikationer.

Avancerad behörighetsstyrning

Användare måste autentisera sig innan tillgång till någon information i RODA. Alla aktiviteter loggas i systemet för full spårbarhet. Behörighetsstyrningen är granulär och kan styras ner på individuella arkivobjekt.

Integrationer med tredjepartsprogram

RODA tillgängliggör all sin funktionalitet via väldokumenterade REST-API. Lämpliga Java-bibliotek finns tillgängliga på GitHub för alla utvecklare som interagerar med RODA via dess Core APIs. Det finns ett flertal verktyg som skapar, transformerar och lämnar in SIP:ar till RODA's ingest-process.

Kvalificerad process för inleverans

RODA stöder inleverans av nytt digitalt material samt tillhörande metadata i flera olika format. Verktyg finns tillgängliga för att skapa SIP:ar i de format som stöds. Arbetsflödet för inleverans kan anpassas i användargränssnittet för att klara av speciella policyer och data.

Och mycket mer...

RODA (Repository of Authentic Digital Records) är ett digitalt arkivsystem för långtidsbevaring av information. Systemet implementerar ett arbetsflöde för inleverans som inte bara validerar SIP:ar, utan kontrollerar också dess innehåll för virus, identifierar format och versioner, extraherar teknisk metadata och konverterar filer till mer arkivbeständiga format.

RODA ger även tillgång till digital information i flera former så som via sök- och navigeringsmöjligheter i tillgänglig metadata. Samt via visualisering och nedladdning av informationen i original, bevarandeformat och spridningsderivat (DIP:ar) direkt i webbgränssnittet.

Administrationsgränssnittet tillåter användare med högre behörighet att hantera och adminsitrera arkivbestånd/informationssamlingar och definiera regler för bevarandeåtgärder och arkivvårdsjobb. All interaktion mellan användare (människor och maskiner) och systemet loggas av säkerhets- och ansvarsskäl.

RODA säkerställer att inlevererad data är autentisk genom att skapa PREMIS-metadata för alla åtgärder som utförs av systemet, sparar information om datats ursprung i arkivmetadatastandarder som EAD eller DC, och säkerställer integritet och tillgänglighet genom att ofta övervaka data och se till att den inte har manipulerats.
