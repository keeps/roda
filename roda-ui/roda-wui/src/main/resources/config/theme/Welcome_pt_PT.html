Bem-vindo ao RODA!

Um repositório digital de código aberto projetado para preservação

O RODA é uma solução completa de repositório digital que oferece funcionalidade para todas as principais unidades funcionais do modelo de referência OAIS. O RODA é capaz de ingerir, gerir e fornecer acesso aos diversos tipos de conteúdos digital produzidos por grandes empresas ou entidades públicas. O RODA é baseado em tecnologias de código aberto e suportado por standards existentes, tais como o Open Archival Information System (OAIS), Metadata Encoding and Transmission Standard (METS), Encoded Archival Description (EAD), Dublin Core (DC) e Metadados de preservação (PREMIS).

Baseado em standards abertos

O RODA segue standards abertos, tais como EAD para metadados de descrição, PREMIS para metadados de preservação, METS para metadados estruturais e vários padrões para metadados técnicos (por exemplo, NISO Z39.87 para imagens digitais).

Independente de fornecedor

O RODA é 100% baseado em tecnologias open-source. Toda a infraestrutura necessária para suportar o RODA é independente do fornecedor. Isso significa que se pode usar as distribuições de hardware e Linux que melhor se adequam às suas necessidades institucionais.

Escalável

A natureza orientada a serviços do RODA permite que o sistema seja altamente escalável, permitindo a distribuição da carga de processamento entre vários servidores. Além disso, o uso de componentes avançados de indexação permite que os serviços de descoberta do RODA sejam distribuídos através de vários servidores num cluster para um desempenho ainda maior.

Ações de preservação incorporadas

As ações de preservação e gestão dentro do RODA são tratadas por um módulo de execução de tarefas. O módulo de execução de tarefas permite que o gestor do repositório execute tarefas de preservação num dado conjunto de dados. As ações de preservação incluem conversões de formatos, verificações de checksum, relatórios (por exemplo, para enviar automaticamente e-mails de aceitação/rejeição do SIP), verificações de vírus, etc.

Autenticidade

O RODA usa metadados de preservação (PREMIS) para criar uma cadeia de confiança entre todas as migrações de formatos e verificações de conteúdo. Os metadados de preservação, em conjunto com o estabelecimento da confiança do ambiente circundante (ISO 16363) asseguram a fiabilidade do serviço e a autenticidade dos registos digitais fechados.

Suporte para vários formatos

O RODA é capaz de ingerir quaisqueres tipos de conteúdo. Os componentes de ação de migração suportam a migração de documentos de texto, imagens raster, base de dados relacionais, vídeo e áudio em formatos normalizados para preservação de longo prazo. Um mecanismo de plug-ins permite que o RODA suporte facilmente migrações de formatos adicionais.

Lida com a rápida mudança da natureza da tecnologia

Os plug-ins e módulo de execução de tarefas permitem uma maneira fácil de adicionar mais funcionalidade ao sistema (por exemplo, novas ações de preservação, alertas, ferramentas, etc.). Além disso, a arquitetura orientada a serviços do RODA permite que os seus componentes sejam atualizados incrementalmente, permitindo um uso heterogêneo de tecnologias em todos os níveis: hardware, SO e aplicações.

Controle de acesso avançado

Os utilizadores devem ser autenticados antes de aceder ao repositório. Todas as ações do utilizador são registadas para responsabilização futura. As permissões são granulares e podem ser definidas ao nível do repositório ou até mesmo ao nível dos objetos individuais.

Integração com outros sistemas

O RODA expõe as suas funcionalidades através de uma API REST bem documentada. Bibliotecas Java estão disponíveis no GitHub para permitir que os programadores interajam com o RODA através das suas APIs principais. Várias ferramentas existem para criar e manipular SIPs e submetê-los ao workflow de ingestão do RODA.

Workflow de ingestão avançado

O RODA suporta a ingestão de novos materiais digitais, bem como metadados associados em vários formatos distintos. São fornecidas ferramentas para permitir que os produtores criem SIPs nos formatos suportados. O workflow de ingestão pode ser personalizado na interface do utilizador para lidar com políticas e dados especiais.

E muito mais...

O RODA é um sistema de repositório digital de longo prazo que implementa um workflow de ingestão que não apenas valida SIPs, mas também verifica o seu conteúdo para a presença de vírus, faz identificação de formatos, extrai metadados técnicos e migra formatos de ficheiro para mais "preserváveis".

O RODA também fornece acesso a informações digitais em várias formas, tais como busca/navegação sobre metadados disponíveis, bem como visualização online e download de originais, formatos de preservação e derivados de disseminação.

As interfaces de administração permitem que os utilizadores do backoffice do repositório façam a gestão dos fundos/coleções e definam regras para ações de preservação. Todas as interações entre utilizadores (humanos e máquinas) e o repositório são registradas por razões de segurança e responsabilidade.

O RODA garante que os dados ingeridos são autênticos, registando em metadados PREMIS todas as ações executadas pelo repositório, regista a proveniência usando metadados standard como EAD ou DC e garante a integridade e a disponibilidade monitorizando com frequência os dados e certificando-se de que não foram adulterados.
